## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add session-scoped dynamic entity resolution that injects the user's recently-played artists and albums into the NLU vocabulary when a session starts. This complements the catalog-based slot types (JF-96.2) with session-specific recency — useful for items played in the last few hours that may not yet be synced to the persistent catalog.

**Why this is a supplement, not the primary mechanism**: Since private skills allow per-user catalogs (JF-96.2), the catalog approach handles the bulk of the library persistently. Dynamic entities add value for:
- Recently played items not yet synced to catalog (catalog sync is daily, sessions are real-time)
- Seasonal/temporary listening patterns
- Reducing latency for the most likely requests in the current session

**How dynamic entities work**:
- On session start (LaunchRequest or first intent), send `Dialog.UpdateDynamicEntities` directive
- Overrides slot type values for the session duration only (max ~100 values)
- No model rebuild needed — purely runtime
- Can include synonyms for phonetic variants

**Implementation approach**:

1. **Session startup hook**:
   - In `LaunchRequestHandler` or as a pipeline interceptor, detect new session
   - Query Jellyfin for user's recently played items (last 24-48 hours, top 50)
   - Build dynamic entity payload with phonetic synonyms

2. **Integration with catalog (JF-96.2)**:
   - Dynamic entities supplement the persistent catalog with session-specific recent plays
   - Dynamic entities override catalog values for the session when both exist
   - If JF-96.2 is not yet implemented, dynamic entities serve as standalone improvement

3. **Performance**:
   - Jellyfin API call on session start adds ~200-500ms latency
   - Cache user's recent items in plugin memory with TTL (e.g., 1 hour)
   - Limit to top 50 items

**Dependencies**: Can be implemented independently, but maximizes value alongside JF-96.2

**Files to create/modify**:
- NEW: `Jellyfin.Plugin.AlexaSkill/Alexa/DynamicEntities/DynamicEntityBuilder.cs`
- MODIFY: `Jellyfin.Plugin.AlexaSkill/Alexa/Handler/LaunchRequestHandler.cs` — inject dynamic entities
- MODIFY: `Jellyfin.Plugin.AlexaSkill/Alexa/Pipeline/` — add interceptor for session start
- NEW: Unit tests for dynamic entity payload generation
