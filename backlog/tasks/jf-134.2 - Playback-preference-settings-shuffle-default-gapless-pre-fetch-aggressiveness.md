## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add playback behavior preferences to `PluginConfiguration` so administrators can tune how audio is delivered to Alexa devices.

**Preferences to expose:**
- `DefaultShuffleEnabled` (bool, default: false) — whether shuffle is on by default when starting playlists/albums
- `GaplessPlaybackEnabled` (bool, default: false) — controls aggressive pre-fetch for gapless audio (related to JF-120)
- `PreFetchCount` (int, default: 1, range: 0-3) — how many upcoming items to pre-fetch in PlaybackNearlyFinished
- `DefaultRepeatMode` (enum: None/RepeatAll/RepeatOne, default: None) — initial repeat state
- `MaxQueueSize` (int, default: 50, range: 10-200) — maximum items in the playback queue

**Implementation**: Add properties to `PluginConfiguration`, render a "Playback Preferences" section in config.html with appropriate input types (checkboxes, number inputs, dropdowns). Handlers read these at execution time rather than using hardcoded values.

**Why**: Playback behavior is subjective — some users want gapless playback, others prefer to save bandwidth. Queue size limits prevent memory issues on large libraries.
<!-- SECTION:DESCRIPTION:END -->
