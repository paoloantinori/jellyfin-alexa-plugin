## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Design and implement a reusable APL list/selection template in `AplHelper.cs` that can be shared by all list-returning intent handlers.

**Context**: The plugin already has `BuildNowPlayingDirective()` and `BuildQueueDirective()` in `AplHelper.cs`. A new `BuildListDirective()` method is needed for handlers that return selectable lists of items (browse results, artist songs, search matches, etc.).

**Design requirements**:
- Dark theme, consistent with existing Now Playing and Queue templates
- Each row: thumbnail/art on the left, title + subtitle on the right
- `TouchWrapper` with `SendEvent` for tap-to-select interaction
- Responsive sizing (use `resources` with viewport conditions, like existing templates)
- Import `alexa-layouts` 1.5.0 (consistent with existing templates)
- APL version 1.9 (consistent)
- Scrollable `Sequence` for lists longer than the viewport
- Method signature: `BuildListDirective(string title, List<ListDisplayItem> items, string token)` returning `AplRenderDocumentDirective?`

**Data model**: Create a `ListDisplayItem` record (similar to existing `QueueDisplayItem`) with:
- `Title` (required)
- `Subtitle` (optional)
- `ArtUrl` (optional)
- `Id` (required — used as SendEvent argument for selection)

**Follow the pattern** of the existing `QueueDocument` in AplHelper.cs (lines 193-286) — same JSON structure with Container → title Text → Sequence of rows.

This task **blocks** all subsequent subtasks that wire APL into handlers.
