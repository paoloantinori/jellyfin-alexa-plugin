## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Wire the reusable APL list template from AplHelper into `BrowseLibraryIntentHandler` and `QueryArtistLibraryIntentHandler`.

**Context**: Both handlers return numbered lists of items as speech-only responses. They should additionally render an APL list on screen-capable devices.

**BrowseLibraryIntentHandler**: Returns up to 5 items (artists, albums, songs, movies, or genres). Each item has a Name and potentially an image. The APL list should show items with thumbnails where available.

**QueryArtistLibraryIntentHandler**: Returns songs or albums by a specific artist. The response can be a full list or partial with count. APL list should show tracks with album art.

**Implementation pattern**:
1. Check `AplHelper.DeviceSupportsApl(context)` before building APL
2. Map handler items to `ListDisplayItem` objects (Title, Subtitle, ArtUrl, Id)
3. Call `AplHelper.BuildListDirective(title, items, token)` to create the directive
4. Add the directive to the response's `Directives` list alongside the speech output
5. Handle `UserEvent` requests from list item taps (route to existing play/select logic)

**Key constraint**: Speech output must remain unchanged. APL is additive — only add the directive, don't modify the SSML.
<!-- SECTION:DESCRIPTION:END -->
