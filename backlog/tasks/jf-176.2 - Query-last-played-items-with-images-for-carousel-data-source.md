## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Create a reusable query method that returns the last ~10 played items with their metadata and image URLs, suitable for feeding the APL carousel.

**Implementation notes:**
- This is similar to `DynamicEntityBuilder.BuildLastPlayedValues()` but needs to return richer data (type, image URL) rather than just slot values
- Consider adding a method to `LaunchRequestHandler` or a shared utility that reuses the 5-minute cache pattern from `DynamicEntityBuilder`
- The query should use `InternalItemsQuery` with `OrderBy = (ItemSortBy.DatePlayed, SortOrder.Descending)` and `IncludeItemTypes` for playable media
- Each result should include: `Id` (Guid), `Name` (string), `Type` (BaseItemKind), `ImageUrl` (from `GetImageUrl()` pattern)
- Apply `ApplyLibraryFilter()` for per-user library access control

**Depends on:** nothing (standalone data query)
<!-- SECTION:DESCRIPTION:END -->

## Final Summary

<!-- SECTION:FINAL_SUMMARY:BEGIN -->
Added `GetRecentlyPlayedItems` as a `private protected static` method on BaseHandler. Queries last 20 played items via InternalItemsQuery (DatePlayed descending), applies library filtering and feature flags (MusicEnabled/VideosEnabled/BooksEnabled), deduplicates by name case-insensitively, takes up to 10 items, maps to ListDisplayItem with subtitle (via AplHelper.GetSubtitle, now internal) and image URLs. Made AplHelper.GetSubtitle internal for reuse. 1636 tests pass.
