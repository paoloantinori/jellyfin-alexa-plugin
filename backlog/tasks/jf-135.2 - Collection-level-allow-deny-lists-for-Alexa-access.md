## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add collection-level permissions that allow administrators to restrict Alexa access to specific Jellyfin collections (e.g., "Music", "Kids Movies", "Audiobooks", "Family Videos").

**Permission model:**
- `CollectionAccessMode` (enum: `AllowAll`, `AllowList`, `DenyList`, default: `AllowAll`)
- `AllowedCollections` (List<string>) — collection names/IDs Alexa can access (used in AllowList mode)
- `DeniedCollections` (List<string>) — collection names/IDs Alexa cannot access (used in DenyList mode)

**Implementation**: Add a `CollectionAccessService` that wraps Jellyfin API calls. When `AllowAll`, no filtering. When `AllowList` or `DenyList`, inject `ParentId` filters into library queries. The config page dynamically fetches the list of Jellyfin collections via the API and renders checkboxes for selection.

**Design choice — AllowList vs DenyList**: AllowList is safer for restricted environments (only expose curated collections). DenyList is easier for permissive environments (block specific sensitive collections). Supporting both gives administrators flexibility.

**Why**: A user may have "Kids Music" and "My Music" as separate collections. They want Alexa to only access "Kids Music" on the family Echo. Collection-level control is the right granularity for this.

## Codebase Findings

**ParentId filtering already exists** in several handlers:
- `PlayAlbumIntentHandler.cs:181,223` — `ParentId = albums[0].Id`
- `PlayPlaylistIntentHandler.cs:159,211` — `ParentId = playlist.Id`
- `QueueContinuationFetcher.cs:70,129` — `ParentId = playlist.Id`

**Implementation approach**: Add `AllowedLibraryIds` / `DeniedLibraryIds` string arrays to PluginConfiguration. Create a `GetAllowedParentIds()` helper on BaseHandler that returns the configured IDs. Handlers inject these into queries via `InternalItemsQuery.AncestorIds` or by filtering results post-query.

**Challenge**: Jellyfin doesn't have a built-in "deny" query filter — only positive filtering via `ParentId`/`AncestorIds`. A deny list requires post-query filtering or querying all libraries and subtracting denied ones.

**Key files to modify:**
- `PluginConfiguration.cs` — add collection ID arrays
- `BaseHandler.cs` — add collection filter helper
- `config.html` — add Collection Access section with library picker
- Search/browse handlers — inject collection filters into queries

**Jellyfin API**: Use `ILibraryManager.GetLibraries()` or `GetItemList(new InternalItemsQuery { IncludeItemTypes = new[] { BaseItemKind.CollectionFolder } })` to enumerate available libraries for the config page dropdown.
