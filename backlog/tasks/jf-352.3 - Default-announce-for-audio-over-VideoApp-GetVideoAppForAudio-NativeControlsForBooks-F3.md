DECISION (2026-07-19, autonomous): No code change. The audio-over-VideoApp path (BuildAudioPlayerResponse -> BuildVideoAppAudioResponse) has no locale available (BuildAudioPlayerResponse is the universal-chokepoint helper with no locale param; threading locale through it is invasive across many callers). Under the audio-announce policy decided in F4 (explicit plays stay silent), no default announce is warranted here -- audio-over-VideoApp is an explicit music play delivered via VideoApp for the scrubber (GetVideoAppForAudio), so it stays silent like other audio plays. AC#2-4 are N/A under that policy. REVERSIBLE: if audio plays should announce (full consistency with video), the clean implementation is to add an optional locale param to BuildAudioPlayerResponse + set a default BuildNowPlayingSpeech when non-null + pass locale from the music handlers -- tracked as a deliberate opt-in if wanted.
