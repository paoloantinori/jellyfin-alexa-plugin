## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Create a SMAPI catalog-based custom slot type populated from the user's Jellyfin library. Because each Jellyfin instance has its own private Alexa skill (unique skill ID), this is effectively per-user — we upload that specific user's artists/albums/songs as catalog values attached to their personal skill.

**Why this is the best option for private skills**: Unlike session-scoped dynamic entities, catalogs persist across sessions, support up to 50,000 values (enough for any personal library), and provide entity resolution with canonical IDs that map directly to Jellyfin item IDs. No aggregation or cross-user concerns.

**How catalog-based slot types work**:
- SMAPI catalog API: create catalog → upload values → create slot type referencing catalog via `CatalogValueSupplier` → interaction model uses the slot type
- Catalogs support up to 50,000 values per slot type
- `ReferencedResourceVersionUpdate` SMAPI jobs auto-sync when catalog updates
- Values can include synonyms for phonetic variants (critical for English names in Italian pronunciation)

**Implementation approach**:

1. **Library sync service** (`Alexa/Catalog/`):
   - On plugin startup or scheduled task: fetch user's artists/albums/tracks from Jellyfin
   - Build catalog values with Italian phonetic synonyms for English names
   - Example: `{"id": "jellyfin_artist_abc123", "name": {"value": "Queen", "synonyms": ["kuin", "i queen", "kueen"]}}`
   - Upload to SMAPI as catalog version

2. **Slot type configuration**:
   - Create `JellyfinArtist` custom slot type referencing the artist catalog
   - Create `JellyfinAlbum` custom slot type referencing the album catalog
   - Optionally `JellyfinSong` for track-level recognition
   - Update `PlaySongIntent`, `PlayAlbumIntent`, `PlayArtistSongsIntent` to use these types
   - Entity resolution maps canonical IDs directly back to Jellyfin entity IDs — no fuzzy search needed for catalog matches

3. **Phonetic synonym generation** (`Alexa/Catalog/PhoneticSynonymGenerator.cs`):
   - English→Italian phonetic mapping: "th"→"t", "w"→"v/u", silent "h"→drop, double consonants→single
   - Generate 2-3 synonyms per English artist/album name
   - Italian names get no extra synonyms (native pronunciation matches)

4. **Automated sync**:
   - Jellyfin scheduled task runs daily/weekly to detect library changes
   - Incremental updates: only add/remove changed items
   - Use SMAPI `ReferencedResourceVersionUpdate` job to propagate to interaction model
   - SMAPI build takes ~15-30s — do this asynchronously

5. **Plugin configuration page**:
   - Show catalog sync status (last sync time, number of items, SMAPI build status)
   - Manual "Sync Now" button
   - Configure sync frequency and which item types to include

**Files to create/modify**:
- NEW: `Jellyfin.Plugin.AlexaSkill/Alexa/Catalog/CatalogManager.cs` — SMAPI catalog lifecycle
- NEW: `Jellyfin.Plugin.AlexaSkill/Alexa/Catalog/PhoneticSynonymGenerator.cs` — Italian phonetic variants
- NEW: `Jellyfin.Plugin.AlexaSkill/Alexa/Catalog/LibrarySyncService.cs` — Jellyfin→catalog sync
- NEW: `Jellyfin.Plugin.AlexaSkill/Alexa/Catalog/CatalogSlotTypeBuilder.cs` — slot type generation
- MODIFY: `Jellyfin.Plugin.AlexaSkill/Alexa/InteractionModel/model_it-IT.json` — reference catalog slot types
- MODIFY: Plugin configuration page — catalog sync controls
- NEW: Scheduled task for automated catalog refresh
- NEW: Unit tests for phonetic synonym generation and catalog payload structure

## Implementation Plan for JF-96.2

### Phase 1: Core Library (no SMAPI dependency)
1. **PhoneticSynonymGenerator** (`Alexa/Catalog/PhoneticSynonymGenerator.cs`)
   - English→Italian phonetic mapping: th→t, w→v/u, silent h→drop, double consonants→single, etc.
   - Generate 2-3 synonyms per English artist/album name
   - Skip Italian names (native pronunciation matches)
   - Unit testable independently

2. **CatalogPayload models** (`Alexa/Catalog/CatalogModels.cs`)
   - `CatalogValue` with id, name.value, name.synonyms
   - `CatalogPayload` with values array
   - ID format: `jellyfin_{type}_{guid}` (e.g. `jellyfin_artist_abc123`)

### Phase 2: SMAPI Catalog Integration
3. **CatalogManager** (`Alexa/Catalog/CatalogManager.cs`)
   - Wraps SMAPI catalog API calls using existing `ManagementApi` base class or direct HttpClient
   - Create catalog, create version with URL upload, list catalogs
   - Store catalog IDs in plugin configuration (per-user)
   - Use presigned URL upload flow for catalog values

4. **LibrarySyncService** (`Alexa/Catalog/LibrarySyncService.cs`)
   - Query Jellyfin library for artists/albums via ILibraryManager
   - Build catalog payloads with phonetic synonyms
   - Upload to SMAPI via CatalogManager
   - Create/update slot types referencing catalogs

### Phase 3: Infrastructure
5. **PluginConfiguration changes**
   - Add `CatalogSyncEnabled` bool
   - Add `ArtistCatalogId` / `AlbumCatalogId` per user (stored in User entity)
   - Add `LastCatalogSync` DateTime

6. **CatalogSyncTask** (`EntryPoints/CatalogSyncTask.cs`)
   - IScheduledTask, runs weekly by default
   - Calls LibrarySyncService
   - Shows in Jellyfin dashboard

7. **Registrator.cs** updates
   - Register CatalogManager, LibrarySyncService, CatalogSyncTask

### Phase 4: Configuration UI
8. **config.html** updates
   - Catalog sync section: enable/disable, last sync time, item counts
   - Manual "Sync Now" button via AJAX to controller endpoint

### Phase 5: Tests
9. **Unit tests** for PhoneticSynonymGenerator (comprehensive)
