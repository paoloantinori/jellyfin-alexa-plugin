## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Add search tuning parameters to `PluginConfiguration` so administrators can control how media search and fuzzy matching behave.

**Preferences to expose:**
- `MaxSearchResults` (int, default: 5, range: 1-10) — maximum items returned in disambiguation prompts
- `FuzzyMatchThreshold` (double, default: 0.7, range: 0.3-1.0) — minimum similarity score to consider a match (lower = more permissive)
- `AutoPlaySingleResult` (bool, default: true) — when search returns exactly one result, play it without asking
- `IncludeTypeInDisambiguation` (bool, default: true) — show media type (album/song/artist) in "did you mean?" prompts

**Implementation**: Add properties to `PluginConfiguration`, render a "Search Preferences" section in config.html. The existing `FuzzyMatchBehavior` (Confirm/AutoPlay) continues to work alongside these new fine-grained controls. `FuzzyMatchThreshold` maps to the FuzzyStrings library's similarity cutoff.

**Why**: Libraries of different sizes need different search tuning. A 100-item library needs permissive matching; a 50,000-item library needs strict matching to avoid false positives.
