DECISION (2026-07-19, autonomous): Audio plays stay SILENT by default. Rationale: music plays (PlaySong/PlayAlbum/PlayArtist/PlayByGenre/...) are frequent; the expected UX is a fast silent start, and announcing 'Now playing X' on every explicit play is verbose. RecommendIntentHandler's announce is an INTENTIONAL exception (recommendation context -- the skill picks the track, so confirming what it chose is useful), not an inconsistency. Video-content launches (movies/episodes/channels) announce (JF-349). This is a coherent per-context policy and is ALREADY IMPLEMENTED -- no code change. AC#2-3 N/A. This subsumes F3 (audio-over-VideoApp). REVERSIBLE: if the user wants all plays to announce for full consistency, implement via an optional locale param on BuildAudioPlayerResponse + a default announce (opt-in config flag recommended given the frequency) -- open a new task if so.
