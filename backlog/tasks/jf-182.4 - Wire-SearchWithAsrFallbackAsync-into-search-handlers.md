## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Wire `SearchWithAsrFallbackAsync` into the handlers that perform `SearchTerm` queries. The main targets are:

1. **PlaySongIntentHandler** — the primary song search. Replace the direct Jellyfin query with a call through `SearchWithAsrFallbackAsync`.
2. **SearchMediaIntentHandler** — unified search across content types.
3. **PlayArtistSongsIntentHandler** — the 4-tier artist fallback chain. Each tier is a separate query type (SearchTerm, NameStartsWith, NameContains) — the ASR fallback should apply to the SearchTerm tier.

## Approach
For each handler, wrap the existing Jellyfin `GetItemsAsync` call in a lambda passed to `SearchWithAsrFallbackAsync`. The handler logic after getting results stays the same.

Write an integration-style unit test for PlaySongIntentHandler that verifies: when searching for "lazy bones" and Jellyfin returns 0 for original query but results for "lazybones", the song is found and played.

Run full test suite to confirm no regressions.
<!-- SECTION:DESCRIPTION:END -->
